## CRITICAL (즉시 수정 필요)

### C-1. API 키가 URL 쿼리스트링에 노출
- **영역**: Rust Backend / Security
- **위치**: AI API 호출 부분
- **문제**: `api_key`가 GET 파라미터로 전송되어 서버 로그, 브라우저 히스토리, 네트워크 모니터링에 노출
- **수정 방향**: Authorization 헤더로 이동 (`Authorization: Bearer <key>`)

### C-2. `expect()` 패닉으로 앱 크래시
- **영역**: Rust Backend
- **위치**: DB 초기화, 앱 초기화 코드
- **문제**: `expect()` 사용으로 에러 발생 시 앱 전체 종료 (panic)
- **수정 방향**: `expect()` → `match` 또는 `?` 연산자로 교체, 에러를 사용자에게 표시

### C-3. `.dmg` 바이너리가 repo에 커밋됨
- **영역**: Build
- **문제**: 대용량 바이너리 파일이 git 히스토리에 포함되어 clone 속도 저하
- **수정 방향**: `.gitignore`에 추가, `git filter-branch` 또는 BFG로 히스토리에서 제거

## HIGH (릴리즈 전 수정 권장)

### 보안 관련

#### H-1. `get_config`가 모든 시크릿을 프론트엔드에 반환
- **영역**: Security / Rust Backend
- **문제**: `get_config` Tauri 커맨드가 API 키, 토큰 등 모든 민감 정보를 프론트엔드로 전달
- **수정 방향**: 프론트엔드에 필요한 필드만 선별적으로 반환하는 별도 커맨드 생성

#### H-2. 자격증명 평문 저장
- **영역**: Security
- **위치**: config.json
- **문제**: API 키가 평문으로 저장됨
- **수정 방향**: OS keychain (macOS Keychain, Windows Credential Manager) 사용 또는 암호화 저장

#### H-3. `aw_api_base` 통한 SSRF 가능성
- **영역**: Security
- **문제**: 사용자가 설정한 `aw_api_base` URL로 서버사이드 요청 발생 → 내부 네트워크 접근 가능
- **수정 방향**: URL 허용 목록(allowlist) 또는 localhost/127.0.0.1만 허용

#### H-4. 버킷 ID 미살균
- **영역**: Security
- **문제**: ActivityWatch 버킷 ID가 살균 없이 URL/경로에 사용 → 경로 조작 가능
- **수정 방향**: 버킷 ID에 영문/숫자/하이픈만 허용하는 정규식 검증 추가

### 안정성 관련

#### H-5. 하드코딩된 `/usr/bin/curl`
- **영역**: Rust Backend
- **문제**: Linux 전용 경로. macOS에서는 `/usr/bin/curl` 위치가 다를 수 있음
- **수정 방향**: `which curl` 또는 `reqwest` 같은 Rust HTTP 라이브러리 사용

#### H-6. HTTP 요청 타임아웃 없음
- **영역**: Rust Backend
- **문제**: 외부 API 호출 시 타임아웃 미설정 → 서버 무응답 시 앱 행(hang)
- **수정 방향**: 30초 정도의 타임아웃 설정

#### H-7. DST 전환 시 `unwrap()` 패닉
- **영역**: Rust Backend
- **위치**: 시간 계산 로직
- **문제**: 서머타임 전환 시 존재하지 않는 시간에 대해 `unwrap()` → 패닉
- **수정 방향**: `chrono`의 `earliest()` 또는 `latest()` 사용

### 프론트엔드 관련

#### H-8. `saveConfig()`가 null 토큰 전송
- **영역**: Frontend
- **문제**: 설정 저장 시 초기화되지 않은 토큰 필드가 null로 전송되어 기존 값 덮어쓰기
- **수정 방향**: null/undefined 필드 필터링 또는 기존 값과 병합

#### H-9. `setInterval` 미해제 (메모리 누수)
- **영역**: Frontend
- **문제**: 주기적 작업의 interval이 해제되지 않아 메모리 누수 발생
- **수정 방향**: 컴포넌트/페이지 언마운트 시 `clearInterval()` 호출

#### H-10. Tauri `listen()` 미해제
- **영역**: Frontend
- **문제**: Tauri 이벤트 리스너가 누적되어 이벤트 중복 처리
- **수정 방향**: `listen()`이 반환하는 unlisten 함수를 cleanup에서 호출

### 배포 관련

#### H-11. `trailingSlash: true` 누락
- **영역**: Landing Page (Next.js)
- **위치**: `apps/web/next.config.ts`
- **문제**: GitHub Pages에서 서브페이지 접근 시 404 발생 가능
- **수정 방향**: `next.config.ts`에 `trailingSlash: true` 추가

#### H-12. OG URL이 GitHub repo URL
- **영역**: Landing Page
- **위치**: `apps/web/src/app/layout.tsx`
- **문제**: Open Graph URL이 배포된 랜딩 페이지가 아닌 GitHub repo를 가리킴
- **수정 방향**: `url`을 `https://junghwayang.github.io/daily-reflect/`로 변경
